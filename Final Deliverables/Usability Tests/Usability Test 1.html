Test Plan for: Voice Geometry Painter Application

Persona Type Tested: Secondary

Context: home
Duration: 30 minutes

Complete the following before starting: Greeting and Consent & User Background Information.

Overall Objectives

  • Evaluate the efficiency/accuracy of the voice recognition engine
  • Evaluate the simplicity/efficiency of the command model structure
  • Identify difficulties in memorizing the commands structure
  • Identify performance issues
  • Evaluate the accuracy/efficiency of providing screen coordinates
Scenario

The user needs to complete a homework involving geometric shapes. He has a set of shapes and needs to find some properties (length, area, angles etc.).

Tasks
  • Task 1: Open application in browser at the specified url (1 minute);
  • Task 2: Set new Whiteboard title (1 minute);
  • Task 3: Draw line (segment) at specific coordinates (1 minute);
  • Task 4: Visualize the properties of the created line (1 minute);
  • Task 5: Clear the whiteboard (30 seconds);
  • Task 6: Draw rectangular triangle at a specific position, having specific lengths for cathetuses (2 minutes);
  • Task 7: View and explain the properties of the created triangle (3 minutes);
  • Task 8: Delete created triangle using "delete" voice command (1 minute);
  • Task 9: Draw a square at specific coordinates, having a specific side length. (2 minutes);
  • Task 10: Change line color (1 minute);
  • Task 11: Draw a circle (2 minutes);
  • Task 12: Delete the square (1 minute);
  • Task 13: Undo last performed operation (30 seconds);
  • Task 14: Save the whiteboard locally (2 minutes);
  • Task 15: Share the whiteboard on Dropbox (2 minutes);
Post-test Questions
  • How difficult it was to memorize the commands structure? (1 - easy, 5 - difficult)
  • Once you knew what you want to draw, how difficult it was to compose the final command phrase? (1 - easy, 5 - difficult)
  • How comfortable did you feel using only voice (no touch)? (1 - least comfortable, 5 - very comfortable)
  • Would you recommend the application to your colleagues? (1 - most probably no, 5 - definitely)
  • What other shapes would you add to the application and why?
  • How fast did the application respond to your commands? (1 - very slow, 5 - instantly)
Notes for Test Coordinator
  • Remind the user that you would like them to 'think aloud' so that you can understand what they are thinking, about the task or the product/application;
  • Read the user the scenario for the test;
  • Read the user the tasks he has to complete. If the tasks are complex, you may want to put a sheet of paper in front of the user with each task written out;
  • Record the user's actions, comments, questions, and body language (write them in the Test Observations section accompanied by video/audio).
  • Don't offer help; let the user attempt to perform the task. If they ask for help reply with: "What do you think you/that would do?" "What do you think that means?".

Other Notes

Test Results for: Voice Geometry Painter Application

Success Criteria

A successful design has been achieved when:

  • 60% of users complete successfully all tasks.
  • 70% of users manage to execute completed tasks in given time.
  • 60% of users have overall positive impression about application.

Summary

  • Two users didn't have a Dropbox account
  • One user drawn a normal triangle instead of and equilateral one, because the speech recognizer missed the "equilateral" keyword
  • One user encountered a misbehavior of "Undo" command
  • One user got high latency in response to the spoken commands
  • Two users needed to repeat some commands because they weren't recognized the first time

Post-test Questions Answers
  • The commands with higher number of parameters were more difficult to remember
  • Once the shape parameters were known (position, size etc.) the command composition did not became less difficult
  • Users felt very comfortable using only voice
  • Most of the users would recommend the application, especially for fun
  • The application response time still needs to be boosted
Test Observations
  • Some users forget to start the speech recognizer before speaking commands
  • Recognition quality varies with the microphone equipment quality
Potential Design Improvements
  • Display shape properties visually on the whiteboard (e.g. triangle side length next to the visual line segment etc.)
  • Add command for moving a shape to other location
  • Start speech recognizer by default when starting the application
